## Cloudflare配置

### 获取API令牌
1. 登录[Cloudflare Dashboard](https://dash.cloudflare.com)
2. 进入 **个人资料** > **API令牌**
3. 创建新令牌，选择 **编辑Cloudflare Workers** 权限
4. 复制令牌并保存

### 部署到Cloudflare Workers

```bash
# 安装Wrangler CLI
npm install -g wrangler

# 登录
wrangler login

# 部署
wrangler deploy
```

### 配置KV存储（可选）
用于存储节点和用户数据

```bash
# 创建KV命名空间
wrangler kv:namespace create "antarctic-fox"
wrangler kv:namespace create "antarctic-fox" --preview

# 在wrangler.toml中配置
[[kv_namespaces]]
binding = "STORAGE"
id = "your-namespace-id"
```

### 配置D1数据库（可选）
用于存储用户账户和配置

```bash
# 创建数据库
wrangler d1 create antarctic-fox

# 在wrangler.toml中配置
[[d1_databases]]
binding = "DB"
database_name = "antarctic-fox"
database_id = "your-database-id"
```

## 本地部署

### 前置要求
- Node.js 16.0.0 或更高版本
- npm 7.0.0 或更高版本

### 安装步骤

```bash
# 克隆仓库
git clone https://github.com/luobikang2/Antarctic-Fox.git
cd Antarctic-Fox

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
# 编辑.env文件，填入你的配置

# 启动开发服务器
npm run dev

# 或启动生产服务器
npm start
```

访问 `http://localhost:3000`

## Docker部署

### 前置要求
- Docker 20.10+
- Docker Compose 1.29+ (可选)

### 使用Docker运行

```bash
# 构建镜像
docker build -t antarctic-fox:latest .

# 运行容器
docker run -d \
  --name antarctic-fox \
  -p 3000:3000 \
  -e PORT=3000 \
  -e JWT_SECRET=your-secret-key \
  antarctic-fox:latest
```

### 使用Docker Compose运行

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  antarctic-fox:
    build: .
    ports:
      - "3000:3000"
    environment:
      - PORT=3000
      - NODE_ENV=production
      - JWT_SECRET=your-secret-key
    volumes:
      - ./data:/app/data
    restart: unless-stopped
```

运行：
```bash
docker-compose up -d
```

## 节点配置

### VLESS节点
最主要的节点类型，支持所有设备。

**配置参数：**
- UUID: 用户标识符
- 域名: Cloudflare绑定的域名
- 加密: none
- 安全: tls
- SNI: 域名

**链接格式：**
```
vless://[UUID]@[域名]:443?encryption=none&security=tls&sni=[域名]&type=ws&host=[域名]&path=/vless
```

### TUIC节点
高性能QUIC协议节点，作为VLESS的补充。

**配置参数：**
- UUID: 用户标识符
- 域名: Cloudflare绑定的域名
- 拥塞控制: cubic

**链接格式：**
```
tuic://[UUID]@[域名]:443?congestion_control=cubic&udp_relay_mode=native
```

### Wireguard节点
直接调用Cloudflare的UDP功能。

**配置参数：**
- PrivateKey: 私钥
- PublicKey: 公钥
- 地址: 虚拟IP地址
- DNS: 1.1.1.1
- Endpoint: Cloudflare节点地址

**配置格式：**
```ini
[Interface]
Address = 10.0.0.2/24
PrivateKey = [私钥]
DNS = 1.1.1.1

[Peer]
PublicKey = [公钥]
AllowedIPs = 0.0.0.0/0
Endpoint = [域名]:51820
```

## API端点

### 认证
- `POST /api/auth/register` - 注册
- `POST /api/auth/login` - 登录
- `GET /api/auth/verify` - 验证令牌

### 节点管理
- `POST /api/nodes/generate-vless` - 生成VLESS节点
- `POST /api/nodes/generate-tuic` - 生成TUIC节点
- `POST /api/nodes/generate-wireguard` - 生成Wireguard节点
- `GET /api/nodes/list` - 获取所有节点
- `DELETE /api/nodes/:id` - 删除节点

### 面板管理
- `POST /api/panel/generate-uuid` - 生成UUID
- `POST /api/panel/config` - 保存配置
- `GET /api/panel/config/:configId` - 获取配置
- `POST /api/panel/theme` - 更新主题
- `POST /api/panel/language` - 设置语言
- `GET /api/panel/status` - 获取面板状态

### 测试
- `POST /api/test/connectivity` - 测试连通性
- `POST /api/test/latency` - 测试延迟
- `POST /api/test/validate-config` - 验证配置

## 故障排除

### 无法连接到Cloudflare
- 检查API令牌是否有效
- 确认账户ID和区域ID正确
- 检查网络连接

### 节点无法连接
- 验证UUID和域名配置
- 确保Cloudflare DNS指向正确的服务器
- 检查防火墙设置

### Docker容器无法启动
- 查看容器日志: `docker logs antarctic-fox`
- 确认端口3000未被占用
- 检查环境变量是否正确设置

## 安全建议

1. **更改默认密码** - 首次登录后立即修改管理员密码
2. **启用HTTPS** - 在生产环境中始终使用HTTPS
3. **定期更新** - 保持应用和依赖项最新
4. **备份数据** - 定期备份D1数据库
5. **限制访问** - 使用防火墙限制对管理面板的访问
